# Changelog

All notable changes to mrclean-claude are documented in this file. The format
follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/) and the
project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

Subsequent entries are generated by [changesets](https://github.com/changesets/changesets);
add one with `npx changeset add` in your PR.

## 1.0.0-rc.4 — 2026-06-01

### Fixed

- **Install stub no longer advertises non-functional config keys.** The
  auto-generated `.mrclean/config.toml` promised `[words]` and `[detection]`
  TOML sections that the config parser never read, so proprietary terms added
  under `[words]` silently failed to redact — a confidentiality footgun for a
  redaction tool. The stub now points custom-term users to the real
  `.mrclean/words.txt` mechanism (with `word|action` overrides), documents only
  parser-backed keys (`dry_run`, `[allowlist]`, `[entropy]`, `[[rules]]`), and
  corrects the repository URL to `github.com/fuzzyqbit/mrclean`. No parser or
  runtime behavior changed.

## 1.0.0-rc.3 — 2026-05-31

### Fixed

- **Self-redaction exemption for mrclean's own MCP tools.** PreToolUse and
  PostToolUse no longer redact the payloads of mrclean's own MCP tools
  (`mrclean_check`/`mrclean_redact`/`mrclean_status`), which previously caused
  the tool surface to redact its own diagnostic output.

## 1.0.0-rc.2 — 2026-05-31

### Fixed

- **Plugin `hooks.json` loader shape.** Wrapped `hooks.json` in a top-level
  `"hooks"` key so the Claude Code plugin loader finds the hook record at
  `.hooks` (matching the `.mcp.json` `mcpServers` shape). Bumped to ship the
  plugin `hooks.json` fix.

## 1.0.0-rc.1 — 2026-05-14

First public release candidate. The published artifact installs as `mrclean-claude`
on npm; the in-session binaries remain `mrclean` and `mrclean-mcp`.

### Added (Phase 1 — Wired Skeleton)

- INST-01..08: Single-command installer (`npx mrclean install`) with idempotent
  settings.json edits, automatic timestamped backup, absolute-path resolution
  (Pitfall #7 defense), uninstall via oldest-backup restoration, project-local
  `.mrclean/` with auto-gitignore, two-bin npm package entry.
- HOOK-01/05/06/07: Four hook event handlers, fail-closed exit-code-2 on
  uncaught exceptions, stdout-only-for-response (HOOK-06), wiring banner on
  SessionStart.
- MCP-01/04: Long-lived stdio MCP server scaffold with three Phase 1 no-op
  tool stubs, single SIGINT/SIGTERM registration site, Zod v4 input validation.
- CFG-01/03: Three-layer config reader (bundled < user-global < project-local).
- AUDIT-03: Audit log append-only contract + gitignore.

### Added (Phase 2 — Live Redaction Layers 1-4 + One-Way)

- DET1-01..04: Layer 1 regex engine — secretlint preset-recommend + vendored
  gitleaks rule pack (SHA 9febafb62, 183 usable rules) + ReDoS-safe worker_threads
  pool.
- DET2-01..03 + DET3-01..03 + DET4-01..03: Layer 2 Shannon entropy with shape
  allowlist; Layer 3 .env value extraction; Layer 4 dirty-word list with
  per-entry action override.
- PH-01..04 + AUDIT-01/02: `<MRCLEAN:TYPE:NNN>` placeholder manager + JSONL
  audit log (hash + fingerprint only, no raw values).
- HOOK-02/03/04: UserPromptSubmit block-with-reason; PreToolUse updatedInput
  substitution; PostToolUse updatedToolOutput (Claude Code >= 2.1.121).
- MODE-01/02: `dry_run = true` first-run mode; one-way redaction default.
- CFG-02/04: 5-axis allowlist schema; `mrclean ignore <fingerprint>` CLI.

### Added (Phase 3 — MCP Tools, Performance Gate, Public Release)

- MCP-02/03: Three production MCP tools — `mrclean_check` (read-only scan),
  `mrclean_redact` (scan + audit + placeholder substitute), `mrclean_status`
  (version + rule_count + mode). No `unredact`, no `disable`, no `add_word`,
  no `config_write` (Pitfall #10 defense).
- PERF-01..03: Vitest assertion gate enforcing UserPromptSubmit p95 <= 100 ms
  on 4 KB and PostToolUse p95 <= 200 ms on 50 KB; CI workflow runs the gate
  on every push and PR; static grep gate enforces compile-once regex pattern.
- DOC-01..03: This README, this CHANGELOG, the THREAT_MODEL.md.
- QA-01..03: >=80% line coverage on src/; integration tests per hook event;
  canary-leak CI guard asserting no fixture-secret value appears in any audit
  entry.

### Deferred to v1.x / v2 (REVMODE, LLM5, POLISH)

- REVMODE-01..03: Reversible-mode (PostToolUse restoration) requires Claude
  Code >= 2.1.121 and a persisted session-state adapter — see REQUIREMENTS.md
  v2 section.
- LLM5-01: Opt-in Layer 5 semantic detection via Claude Haiku 4.5.
- POLISH-01..03: Session-report command, cross-session deterministic
  placeholders, encrypted at-rest map.
