# Quick Task 260601-1sw: `mrclean init` CLI subcommand + `/mrclean:mrclean-init` slash command

## Why (decision record)

User wanted a slash command to initialize the project mrclean env (create `.mrclean/`,
config stub, and seed `words.txt`). Original ask was an MCP tool (`mrclean_init`), but
the MCP surface enforces a **threat-model invariant (MCP-03)**: exactly 3 tools, all
read-only, no disk writes — `tests/mcp/tools-list.test.ts` bans write/config tool names
to close a prompt-injection surface (server.ts:13-16). A write-capable MCP tool would
break that invariant.

Chosen path (#1): keep the MCP surface untouched. Put the deterministic logic in a CLI
subcommand `mrclean init`, and ship a thin slash command that shells it. Same UX, no
security regression.

## Tasks

1. `src/install/init-project.ts` (new): `WORDS_TXT_STUB` (comment-only seed) + `runInit({cwd})`.
   - Reuse `createProjectDir` (dir + config stub, idempotent, no clobber).
   - Seed `words.txt` only if absent (no clobber).
   - Call `addGitignoreEntries` so `.mrclean/` (incl. words.txt proprietary terms) is gitignored.
   - Print a per-artifact summary (created vs already-present); return structured `InitResult`.
   - verify: unit tests below pass.
   - done: helper idempotent, seed parses to ZERO active word entries.

2. `src/cli.ts`: register `init` subcommand → lazy-import `runInit`.
   - verify: `program.commands` contains `init`.

   body instructs Claude to run it and explain words.txt to the user.
