# Quick Task 260601-2fj: uninstall surgically removes only mrclean entries (no wholesale restore)

## Bug

`runUninstall` (src/install/index.ts) restored the *entire* settings.json / .claude.json
from the oldest mrclean backup snapshot. That clobbers any unrelated change the user made
after install. Real impact this session: uninstall dropped an unrelated plugin enablement
and re-enabled a plugin the user had disabled. For a tool editing shared global config,
full-restore is destructive.

Surgical removers already exist and are correct: `removeHookEntries` (filters `_mrclean`
marker), `removeMcpServerEntry` (deletes the `mrclean` key). Uninstall just wasn't using
them — it preferred backup-restore, falling back to surgical only when no backup existed.

## Fix

1. `src/install/index.ts`: `runUninstall` calls `removeHookEntries` + `removeMcpServerEntry`
   + `removeGitignoreEntries` directly. Delete the `restoreOrRemoveHooks`/`restoreOrRemoveMcp`
   backup-restore helpers and the now-unused `listMrcleanBackups`/`restoreFromBackup` import.
   Install still writes backups (safety) — we just never wholesale-restore them.
   - verify: unrelated settings survive uninstall; mrclean entries gone.

2. `tests/install/uninstall-roundtrip.test.ts`: the byte-identity assertion encoded the
   buggy behavior. Replace Test 1 with a regression test: pre-install unrelated keys +
   a post-install unrelated change both survive uninstall, and mrclean hook entries are
   gone. Keep the .gitignore and .mrclean-retained tests (gitignore removal was always
   surgical).
   - verify: new test fails on old code (red), passes on new (green); full suite green.

## Non-goals / scope

- Not pruning the benign empty `hooks:{event:[]}` / `mcpServers:{}` residue —
  that is removeHookEntries' existing documented contract (settings.test.ts) and is
  semantically a no-op for Claude Code. Out of scope to avoid churning unrelated tests.

## must_haves

- truths: uninstall preserves all non-mrclean settings, including post-install changes.
- artifacts: src/install/index.ts, tests/install/uninstall-roundtrip.test.ts.
- key_links: reuses removeHookEntries (settings.ts), removeMcpServerEntry (mcp-config.ts).
