<objective>
Add the operator-facing `[pii]` configuration sub-table — OFF by default — to the existing three-layer config reader, so Phases 5-7 have a stable config surface to gate the regex and NER lanes against. Per-entity action policy is expressible (checksum-validated entities like SSN/credit-card may `block`; other PII defaults to `warn`/`audit`). With no `[pii]` table present, behavior is byte-identical to v1.

Purpose: PII-03 is the contract that makes PII opt-in. Locking the config shape now (master switch, regex sub-lane, ner sub-lane, per-entity toggles, action policy, confidence) before any detector exists means Phase 5/6 wire detection behind a frozen, tested config — no schema churn mid-implementation. The off-by-default + absent-table-equals-v1 guarantee is the load-bearing "core secret tool is provably unchanged" requirement.

Output: `MrcleanPiiConfig` interface in `shared/types.ts`; `[pii]` parser + validator + merge semantics in `config/index.ts`; frozen `pii` defaults (off) in `config/defaults.ts`. No detector reads this config in Phase 4 — config surface only.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| .mrclean/config.toml → MrcleanConfig | Untrusted operator file crosses into the typed config; malformed input must fail with a structured error, never silently enable PII or corrupt secret config. |
| pii defaults → core secret path | The off-by-default master switch must guarantee absent-[pii] == v1; a regression here would silently change core behavior. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-01 | Tampering | mergeConfigs pii path | mitigate | Deep-equal test asserts absent-[pii] produces byte-identical-v1 config; a merge bug that leaked pii state into the secret path would fail it. |
| T-04-02-02 | Elevation of privilege | per-entity actions map | mitigate | actions values validated against {block,warn,audit}; an operator cannot inject an unknown action token. Default map keeps non-checksum PII at warn/audit (advisory), honoring the "secrets remain the only default hard gate" decision. |
| T-04-02-03 | Denial of service | validatePiiConfig on hostile TOML | accept | smol-toml already bounds parse; validators are O(entities) over a small fixed key set. No untrusted regex compiled here. Low risk. |
| T-04-02-SC | Tampering | npm/pip/cargo installs | mitigate | This plan adds NO package-manager installs (config-only). No package legitimacy gate required. |
</threat_model>
