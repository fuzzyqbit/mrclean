<objective>
Lock two expensive-to-reverse architectural decisions: (1) MODEL-01 — declare the ML dependency subtree (`@huggingface/transformers`, `onnxruntime-node`) as `optionalDependencies` so a failed native build NEVER breaks the core secret tool; and (2) PIISEC-03 — document and ENFORCE the v2.0 scope fence (no cloud PII APIs, no model-facing unredact tool, no Presidio Python sidecar in the default distribution), extending the existing MCP-03 forbidden-tool invariant test to ban PII-write/unredact tools.

Purpose: optionalDependencies is the dependency-layer guarantee behind "core installs and runs with zero ML deps present" (Pitfall 2). The scope fence (Pitfall 11) prevents the milestone drifting into "a worse Presidio in Node." Both are cheap to assert now and a structural retrofit later. No ML code is added — only the dependency declaration, the documented fence, and the enforcing test.

Output: `optionalDependencies` block in package.json (ML deps NOT in `dependencies`); `docs/SCOPE-FENCE.md` with the bans + a transition checklist; a THREAT_MODEL.md non-defense entry pointing at the fence; the MCP-03 forbidden-tool list extended with PII-write/unredact names; a test that the core package's `dependencies` contains no ML dep.
</objective>
