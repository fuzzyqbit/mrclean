<objective>
Make raw-PII leakage into audit logs and error/diagnostic output structurally impossible, and prove it with a regression test. Delivers PIISEC-01: one central `sanitizeForOutput()` chokepoint on the error/stderr path (D-03/D-04), the two known raw-error leak vectors (`supervisor.ts`, `failclosed.ts`) routed through it, plus a leak-grep integration test that feeds synthetic PII through the full NER-on pipeline and asserts no raw value reaches `audit.jsonl` OR stderr — including three deliberately-triggered failure paths (D-01/D-02).

Purpose: A security tool that leaks the very PII it claims to scrub — via an error message — is the worst-case trust failure. Defense-in-depth (runtime chokepoint + test) per CONTEXT §specifics.
Output: `src/shared/sanitize-output.ts`, its unit test, edits to `supervisor.ts` + `failclosed.ts`, and `tests/audit/pii-canary-leak.test.ts`.
</objective>
