<objective>
Add a stable, machine-readable best-effort flag to NER findings wherever they surface programmatically, satisfying SC-3 ("the probabilistic asterisk on NER findings is visible wherever PII results surface") without per-finding visual noise. Delivers the D-06 slice of PIISEC-02: an always-emitted `bestEffort: z.boolean()` field on the per-finding DTO in both MCP tools, set `true` only for the probabilistic NER lane (`source === 'pii-ner'`) and `false` for every deterministic finding.

NOTE — PIISEC-02 is split across two plans by design (see ROADMAP §Phase 7): this plan (07-02) owns ONLY the D-06 machine-readable flag; the user-facing honest-framing copy (D-05 once-per-output disclaimer on README/doctor/banner+CLI), the framing-content stance (D-07 README PII section + SCOPE-FENCE alignment), and the banned-phrase/copy-drift CI test (D-08) are delivered in 07-03 (wave 2). This split is explicit, not a scope reduction — every locked PIISEC-02 decision is covered across the 07-02 + 07-03 plan set.

Purpose: Programmatic MCP consumers must distinguish the deterministic secret/regex-PII guarantee from the best-effort NER recall aid. A typed boolean (mirroring the existing `nerStatus` enum precedent) can never carry matched PII, unlike free text (D-06).
Output: `bestEffort` field added to `findingSchema` + `toFindingDTO` in `check.ts` and `redact.ts` (kept as exact mirrors), with extended unit tests in `tests/mcp/check.test.ts` + `tests/mcp/redact.test.ts`.
</objective>
