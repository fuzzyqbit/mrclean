<objective>
Frame the PII/NER layer honestly everywhere it surfaces, and lock that framing against future copy-drift. Delivers PIISEC-02: a stable machine-readable `bestEffort` boolean on NER findings in MCP `check`/`redact` structuredContent (D-06), a once-per-output best-effort disclaimer on README + `mrclean doctor` + the CLI/banner line (D-05/D-07), and a CI grep test (D-08) that fails the build if compliance/guarantee language appears in user-facing strings.

Purpose: Secrets are the deterministic guarantee; PII/NER is a best-effort recall aid whose false negatives CAN leak. Copy must never blur the two into "mrclean redacts your PII." The disclaimer points users to `words.txt` + deterministic layers as the real must-not-leak lever (CONTEXT §specifics, D-07).
Output: edits to check.ts/redact.ts (+ unit test), doctor checks, banner, README PII section, and a new banned-phrase CI test.
</objective>
