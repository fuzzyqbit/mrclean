<objective>
Build the model-acquisition infrastructure the Phase 6 NER lane depends on — fully testable
WITHOUT any inference. The default model (`Xenova/bert-base-NER` `onnx/model_int8.onnx`, 108 MB)
lazy-downloads to a stable absolute `~/.mrclean/models/` cache (NEVER cwd-relative `./.cache`),
is SHA-256-pinned and verified on load (refused on mismatch), supports an offline/air-gapped
side-load path (`mrclean pii fetch-model --from <path>`), and is reported by `mrclean doctor`.
The default PII-off `npx` cold path never imports ML deps and never touches the network.

Purpose: Ship the download/cache/integrity/side-load plumbing as pure infra so Phase 6 can wire
NER inference against a verified, stable cache. Output: `src/model/constants.ts`,
`src/model/model-cache.ts`, a `checkModelCache` doctor check, the `mrclean pii fetch-model`
CLI subcommand, and unit tests with a mocked download.

Implements MODEL-02 (lazy download to stable cache, cold path never loads ML deps) and
MODEL-03 (pinned SHA-256 verify, side-load, doctor integrity report).

NOTE: Task 0 is a BLOCKING Wave-0 setup task — it computes the real PINNED_MODEL_SHA256 from
the live model file and requires a human checkpoint to confirm the network fetch. No
implementation task can complete until the real hash is committed (RESEARCH Pitfall 6).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| huggingface.co → local cache | Remote model bytes downloaded over the network into ~/.mrclean/models/ |
| operator --from path → local cache | Air-gapped operator-supplied file copied into the cache |
| cwd → cache dir resolution | A malicious cwd could plant a poisoned ./.cache if the path were cwd-relative |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02-01 | Tampering | downloadModel / sideLoadModel (model file integrity) | mitigate | SHA-256 pinned in src/model/constants.ts (real hash from Task 0); verified on download AND side-load BEFORE the file is moved into MODEL_CACHE_PATH; fail-closed (unlink temp, throw) on mismatch — never auto-trust a re-download. |
| T-05-02-02 | Tampering | cache dir resolution | mitigate | MODEL_CACHE_PATH is built from os.homedir() + path.join → absolute ~/.mrclean/models/, NEVER cwd-relative ./.cache (RESEARCH Pitfall 1 / Pitfall #2 from threat brief). A malicious cwd cannot plant a poisoned cache. |
| T-05-02-03 | Elevation of Privilege / Tampering | sideLoadModel --from path | mitigate | The --from path is resolved to absolute, asserted to exist and be a regular file before any read; verified by SHA-256 before acceptance — path-traversal / arbitrary-file load is rejected by the existence+regular-file+hash gate. |
| T-05-02-04 | Denial of Service | model download on hook cold path | mitigate | model-cache.ts imports zero ML deps; the CLI fetch-model action uses a DYNAMIC import; the default PII-off npx cold path (install/hook/doctor-absent) never imports model-cache or touches the network (MODEL-02). |
| T-05-02-05 | Information Disclosure | doctor model check output | accept | checkModelCache returns only model name + status + a generic detail string; no raw model bytes, paths outside home, or secrets are emitted. |
| T-05-02-SC | Tampering | npm/pip/cargo installs | mitigate | No NEW packages introduced (RESEARCH Package Legitimacy Audit: pass-through; @huggingface/transformers + onnxruntime-node were reviewed and declared optionalDependencies in Phase 4-03). model-cache.ts uses only Node stdlib + fetch. No install task; no [ASSUMED]/[SUS] package checkpoint required. Task 0's network fetch is a one-time hash-pinning download (a blocking-human checkpoint), not a package install. |
</threat_model>
