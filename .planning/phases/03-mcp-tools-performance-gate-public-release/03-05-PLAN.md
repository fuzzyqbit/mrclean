release.yml structure (RESEARCH §Pattern 4 — LOCKED):
```yaml
name: Release
on:
  push:
    branches: [main]
concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: false   # do NOT cancel in-progress releases
jobs:
  release:
    runs-on: ubuntu-latest
    timeout-minutes: 20
    permissions:
      id-token: write       # REQUIRED for npm --provenance via OIDC
      contents: write       # REQUIRED for changesets to create/update PRs
      pull-requests: write  # REQUIRED for changesets to create/update PRs
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0    # changesets needs full history
      - uses: actions/setup-node@v4
        with:
          node-version: '20.x'
          registry-url: 'https://registry.npmjs.org'
          cache: 'npm'
      - run: npm ci
      - run: npm run build
      - run: npm test
      - name: Create Release PR or Publish
        id: changesets
        uses: changesets/action@v1
        with:
          publish: npm run release   # invokes `npm run build && changeset publish --provenance --access public` (set in package.json by plan 03-00)
          commit: "chore: version packages"
          title: "chore: version packages"
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          NPM_TOKEN: ${{ secrets.NPM_TOKEN }}
          NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}   # actions/setup-node uses this var per RESEARCH §Pitfall 6
      - name: Trigger release-smoke if published
        if: steps.changesets.outputs.published == 'true'
        run: echo "::notice::Published — release-smoke workflow will follow"
```

release-smoke.yml (CONTEXT.md §"Clean-machine reproducibility"):
```yaml
name: Release Smoke
on:
  workflow_run:
    workflows: ["Release"]
    types: [completed]
    branches: [main]
jobs:
  smoke:
    if: ${{ github.event.workflow_run.conclusion == 'success' }}
    runs-on: ubuntu-latest
    timeout-minutes: 15
    steps:
      - uses: actions/setup-node@v4
        with:
          node-version: '20.x'
      - name: Wait for npm registry propagation (90s)
        run: sleep 90
      - name: Install the latest published mrclean-claude
        run: |
          set -e
          npm install -g mrclean-claude@latest
          which mrclean
          which mrclean-mcp
          mrclean --version
      - name: Phase 1 success criterion #1 — install creates settings.json entries
        run: |
          set -e
          mkdir -p $HOME/.claude
          # Create empty Claude Code settings to simulate fresh machine
          echo '{}' > $HOME/.claude/settings.json
          echo '{}' > $HOME/.claude.json
          mkdir -p /tmp/test-project && cd /tmp/test-project
          mrclean install
          # Assert hook + mcp wiring present
          grep -q "mrclean" $HOME/.claude/settings.json || (echo "::error::settings.json missing mrclean entry"; exit 1)
          grep -q "mrclean" $HOME/.claude.json || (echo "::error::.claude.json missing mrclean-mcp entry"; exit 1)
      - name: Phase 1 success criterion #2 — doctor reports green
        run: |
          set -e
          cd /tmp/test-project
          # Use the test escape hatch from STATE.md (MRCLEAN_TEST_FAKE_CLAUDE_VERSION) since no real claude binary is in PATH
          MRCLEAN_TEST_FAKE_CLAUDE_VERSION=2.1.121 mrclean doctor || (echo "::error::doctor reports failure"; exit 1)
      - name: Phase 2 success criterion — MCP tools/list returns the three production names
        run: |
          set -e
          # Spawn mrclean-mcp and send an initialize + tools/list via stdin pipe
          cd /tmp/test-project
          OUTPUT=$(node -e "
            const { spawn } = require('node:child_process');
            const child = spawn('mrclean-mcp', { stdio: ['pipe', 'pipe', 'pipe'] });
            const messages = [
              { jsonrpc: '2.0', id: 1, method: 'initialize', params: { protocolVersion: '2025-11-25', capabilities: {}, clientInfo: { name: 'smoke', version: '0.0.0' } } },
              { jsonrpc: '2.0', method: 'notifications/initialized' },
              { jsonrpc: '2.0', id: 2, method: 'tools/list', params: {} },
            ];
            messages.forEach(m => child.stdin.write(JSON.stringify(m) + '\\n'));
            let buf = '';
            child.stdout.on('data', d => buf += d.toString());
            setTimeout(() => { child.kill(); console.log(buf); }, 4000);
          ")
          echo "$OUTPUT" | grep -q '\"mrclean_check\"' || (echo "::error::mrclean_check absent from tools/list"; exit 1)
          echo "$OUTPUT" | grep -q '\"mrclean_redact\"' || (echo "::error::mrclean_redact absent from tools/list"; exit 1)
          echo "$OUTPUT" | grep -q '\"mrclean_status\"' || (echo "::error::mrclean_status absent from tools/list"; exit 1)
          echo "$OUTPUT" | grep -q '\"mrclean_unredact\"' && (echo "::error::FORBIDDEN tool mrclean_unredact present!"; exit 1) || true
          echo "$OUTPUT" | grep -q '\"sanitize\"' && (echo "::error::FORBIDDEN deprecated tool sanitize present!"; exit 1) || true
          echo "Smoke test passed: tools/list contains exactly the three production tools, no forbidden tools."
      - name: Phase 2 hook canary — UserPromptSubmit with AWS-shape blocks the prompt
        run: |
          set -e
          cd /tmp/test-project
          # Pipe a JSON hook payload with a Stripe key into the hook handler
          # (AWS doc-placeholder values like AKIAIOSFODNN7EXAMPLE are allowlisted per Phase 2 SUMMARY)
          PAYLOAD='{"hook_event_name":"UserPromptSubmit","session_id":"smoke-session","transcript_path":"/dev/null","cwd":"/tmp/test-project","prompt":"my key is sk_live_testABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789abcdef"}'
          RESPONSE=$(echo "$PAYLOAD" | mrclean || true)   # exit 2 is expected for block; capture body
          echo "$RESPONSE" | grep -q '"decision":"block"' || (echo "::error::hook did not block on Stripe key"; exit 1)
          echo "Hook smoke test passed: blocked on stripe-shape secret."
```

Constraints:
- release-smoke uses `workflow_run` trigger so it only fires AFTER Release.yml completes successfully. This is the canonical pattern for post-publish smokes per RESEARCH §Pattern 4 + GitHub Actions docs.
- The 90-second `sleep` allows npm registry propagation; otherwise `npm install -g` may pull the OLD version.
- The smoke uses MRCLEAN_TEST_FAKE_CLAUDE_VERSION (per STATE.md decision) since no real claude binary lives on the runner.
- The hook smoke uses Stripe live-key format (not AWS) per Phase 2 02-05 SUMMARY (AKIA*EXAMPLE is allowlisted).

```markdown
# Release process

## First publish (manual, one-time)

The 1.0.0-rc.1 first-publish runs from the maintainer's local machine because
`npm publish --provenance` requires a GitHub Actions OIDC token (RESEARCH §Pitfall 5).
Subsequent versions auto-publish via the Release workflow with full provenance.

Prerequisites:
- npm account with publish rights for `mrclean-claude`
- 2FA device if the account has 2FA enabled (RESEARCH §OQ-4)
- Green main branch (test.yml + perf.yml + canary-leak.yml all passing)

Steps:
1. `npm whoami` — confirm you are logged in. If not: `npm login`.
2. `git checkout main && git pull`
3. `npm ci`
4. `npm run build`
5. `npm test` — must pass
6. `npm run test:coverage` — must pass
7. `npm pack --dry-run` — review the tarball file list; must match package.json#files
8. `npm publish --access public` — NOTE: NO `--provenance` flag for the local publish
9. Verify: `npm view mrclean-claude` shows version 1.0.0-rc.1
10. `git tag v1.0.0-rc.1 && git push --tags`

## Subsequent publishes (automated via changesets/action)

Per-PR contributor flow:
- Add a changeset: `npx changeset add` — prompts for bump level + summary
- Commit the generated `.changeset/<slug>.md`
- Open a PR, get it reviewed, merge to main

Post-merge:
- The Release workflow runs on push to main.
- changesets/action sees the new `.changeset/*.md` file and opens a "Version Packages" PR.
- A maintainer reviews the version-PR; the PR will update package.json#version + CHANGELOG.md.
- Merge the version-PR.
- The Release workflow runs again. This time there are no pending changesets, so it runs the publish step.
- `npm run release` (defined in package.json) executes: `npm run build && changeset publish --provenance --access public`.
- The release-smoke workflow fires automatically on the Release workflow's successful completion.

## 1.0.0-rc.1 → 1.0.0 transition

Special case: the initial changeset `.changeset/initial-release.md` is shipped in
the repo (created by plan 03-05). The FIRST run of the Release workflow on main
consumes it and opens a version-PR bumping to 1.0.0. After merge, the next Release
workflow run does the 1.0.0 publish.

## Rollback

If a published version is broken:
- `npm dist-tag rm mrclean-claude latest` — un-default the broken version
- `npm dist-tag add mrclean-claude@<previous-good-version> latest` — restore a good default
- npm policy forbids unpublishing after 72 hours; for emergency rollbacks within 72 hours: `npm unpublish mrclean-claude@<version>`
- For provenance-attested broken versions: see https://docs.npmjs.com/policies/security
```

Provenance behavior summary (RESEARCH §Pattern 5):
- Provenance requires id-token: write workflow permission + npm 9.5+ + GitHub-hosted runner.
- Local publishes (Step 8 above) OMIT --provenance. The published package will not have sigstore attestation on 1.0.0-rc.1.
- 1.0.0 onward (automated via changesets) WILL have provenance.
- Users can verify: `npm audit signatures mrclean-claude`.
